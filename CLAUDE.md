Project Blueprint: The Royal Wedding Decree (Digital Experience)1. Project OverviewA high-end, mobile-first web experience featuring a 3D "Wax Seal" opening animation, immersive video backgrounds, and interactive event details.Primary Goal: Mobile-optimized, high-performance transitions.Architecture: Next.js (Static Site Generation - SSG).Styling: Tailwind CSS + Framer Motion.3D Engine: Three.js (via React Three Fiber).2. Recommended Tech StackLayerTechnologyReasonFrameworkNext.js 15 (App Router)Best-in-class performance and image/video handling.3D/PhysicsReact Three Fiber (R3F)To render the 3D scroll and the wax seal interaction.AnimationsFramer MotionFor the "unfolding" paper effect and scroll-triggered text.IconsLucide ReactClean, lightweight icons for location and timeline.FormsReact Hook FormFor the attendance (RSVP) section.3. Visual & Interaction FlowThe Landing (The Seal): A 3D rendered old-world scroll closed with a red wax seal. The user must tap the seal.The Reveal: Upon clicking, the seal "breaks" (physics effect), and the scroll unfurls. The camera zooms "into" the paper, transitioning into the main UI.The Hero: A background video of the couple with a low-opacity parchment overlay. Names and dates appear with elegant typography.The Scroll: Content fades in using "Reveal on Scroll" patterns.The Map & RSVP: Interactive elements for logistics and a functional form.4. Detailed Component StructureSection A: The 3D Wax Seal (/components/Hero3D)Library: @react-three/fiber and @react-three/drei.Assets: A .glb model of a wax seal and a parchment texture.Logic: Use useFrame to create a slight "floating" animation. On click, trigger a GSAP or Framer Motion sequence to scale the seal to 0 and expand the "paper" div.Section B: Hero & Countdown (/components/Countdown)Visuals: next/video for high-performance background playback.Logic: A custom hook useCountdown(weddingDate) to calculate Days/Hours/Mins/Secs.Section C: The Timeline (/components/Timeline)Visuals: A vertical dashed line. Icons represent: Ceremony -> Photos -> Dinner -> Party.Interaction: Use Framer Motion whileInView to animate each milestone from the side.Section D: Location (/components/Venue)Visuals: An iframe embedding Google Maps with a custom-styled card overlay containing the address and "Open in Maps" button.Section E: RSVP Form (/components/RSVP)Fields: Name, Email, Phone, Attendance Status (Yes/No), plus a "Guest Count" dropdown.Integration: Since it's a static site, use Formspree or a simple Resend API route in Next.js to handle the emails without a full backend.5. Implementation Instructions for the AI AgentSystem Prompt for AI Agent:"Build a Next.js 15 project using Tailwind CSS and Framer Motion. The site must be strictly mobile-first.Key Tasks:Setup a canvas using React Three Fiber for the opening 'Wax Seal' interaction.Implement a 'Parchment' CSS texture for the background of all sections.Create a smooth scroll experience where sections 'unfold' like paper as the user scrolls.Ensure the background video is muted, plays inline, and has a 'fallback' image for low-battery modes.Use Google Fonts: 'Pinyon Script' for names and 'Montserrat' for body text.Optimize all assets for mobile (WebP/WebM formats)."6.
